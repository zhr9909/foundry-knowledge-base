# Progress Update (2026-07-10)

## 1. Current Product State

The project has moved from the original plain HTML/CSS/JS frontend toward a Vue-based interactive knowledge-base product.
The main user-facing workflow is now:

1. Open the web app at `http://127.0.0.1:8000/static/index.html#/`
2. Ask a metal material or casting-process question.
3. The gateway streams progress logs and answer data.
4. The RAG agent retrieves relevant handbook chunks, reranks context, calls the LLM, and returns an answer with citations.
5. Conversation history is available after login and can restore previous messages.

The current running services are:

| Port | Service | File | Role |
|---|---|---|---|
| 8000 | Orchestrator Gateway | `scripts/gateway.py` | Static frontend, auth, history, SSE entrypoint |
| 8001 | Generate/RAG Agent | `scripts/agent_rag.py` | RAG pipeline and answer generation |
| 8002 | Search Agent | `scripts/agent_search.py` | Search endpoint and retrieval support |

### Direct LLM API Call

The previous LLM configuration depended on a local proxy at `127.0.0.1:15721`, which caused frontend answers to fail with:

```text
AI 调用失败 (HTTP 502)
```

The fix was to bypass that local proxy and call the LLM provider API directly.

Changes made:

| File | Change |
|---|---|
| `scripts/agent.py` | Added local `.env` loading for LLM config. |
| `scripts/agent.py` | Replaced duplicated hardcoded `127.0.0.1:15721/v1` settings with environment variables. |
| `scripts/agent.py` | Added `trust_env=False` to `httpx.Client(...)` calls so Python does not accidentally use system proxy settings. |
| `scripts/.env` | Stores local LLM API config and key; this file is ignored by Git. |

Important security note:

- `scripts/.env` is ignored by `.gitignore`.
- The API key is not committed to Git.
- Do not print or paste the key into documentation, logs, or committed code.

Validation performed:

| Check | Result |
|---|---|
| Direct provider request from PowerShell | `200 OK` |
| Python `httpx` request with `trust_env=False` | `200 OK` |
| `8001 /chat/stream` | Returned generated answer successfully |
| `8000 /chat/stream` | Returned generated answer successfully after gateway restart |
| HTTP 502 regression | Fixed after restarting both `8001` and `8000` |

## 3. Operational Notes

### Start and Restart

The web app depends on all three ports:

```text
8000 Gateway
8001 Generate/RAG Agent
8002 Search Agent
```

If LLM configuration changes, restart both:

1. `8001` Generate/RAG Agent
2. `8000` Gateway

Reason: `gateway.py` imports the RAG logic directly for the SSE path, so restarting only `8001` may still leave `8000` with stale in-memory config.

### Website URL

Open:

```text
http://127.0.0.1:8000/static/index.html#/
```

If the page shows old layout or old errors, refresh the page and start a new conversation. Old failed messages in history will remain as historical records.

## 4. Next Development Ideas

The next stage should make the app feel less like a generic chat wrapper and more like an industrial AI knowledge workbench.

### P0: Knowledge Graph / Mind Map

Goal: after asking about a material or process, the system can generate a structured visual map.

Example for `铝合金6061`:

- Composition
- Heat treatment states
- Mechanical properties
- Application scenarios
- Related alloys
- Relevant handbook chapters
- Citation nodes

Suggested UX:

- Right-side panel or dedicated graph page.
- Nodes can be clicked to reveal source snippets and citations.
- The graph can be generated from the current answer and retrieved context.

Suggested technical path:

1. Add a graph extraction step after answer generation.
2. Return graph JSON in addition to answer text and citations.
3. Render with a frontend graph library.
4. Later support Mermaid export or image export.

### P1: Citation Source Cards

Current citations work, but the experience can be more professional.

Recommended additions:

- Citation cards with page, section, source type, and relevance score.
- Click citation to open the PDF viewer at the corresponding page.
- Highlight the quoted or retrieved source snippet.
- Show whether the source is a table, text chunk, or image-derived content.

### P1: Structured Answer Modes

Different question types should produce different layouts:

| Question type | Desired output |
|---|---|
| Material property | Summary + property table + source cards |
| Heat treatment | Temperature range + process steps + risk notes |
| Casting process | Process flow + parameter ranges + defect prevention |
| Comparison | Side-by-side comparison table |
| Defect diagnosis | Cause tree + corrective actions |

This can be implemented as a lightweight intent classifier before rendering.

### P1: Conversation Management

Improve the current history system:

- Rename conversation.
- Delete conversation.
- Pin important conversations.
- Search conversations.
- Group conversations by project or material.

### P2: Project Spaces

Add project-level organization:

- One project can contain conversations, saved answers, citations, and generated graphs.
- Example spaces:
  - `铝合金项目`
  - `不锈钢热处理项目`
  - `铸造缺陷诊断项目`

This is useful for long-term industrial workflows, not just one-off Q&A.

### P2: Saved Knowledge Cards

Allow users to save useful answer fragments into reusable cards:

- Material card
- Process card
- Defect card
- Parameter card
- Citation card

These cards can later feed the knowledge graph and project report features.

### P2: Material Comparison

Support prompts such as:

```text
6061 和 6082 的力学性能、焊接性、耐腐蚀性有什么区别？
```

Expected output:

- Comparison table
- Recommended use cases
- Advantages and tradeoffs
- Source-backed citations

### P2: Defect Diagnosis Assistant

Support casting defect workflows:

```text
铸件出现气孔，可能是什么原因？
```

Expected output:

- Possible causes
- Evidence questions to ask next
- Process corrections
- Inspection suggestions
- Related handbook citations

### P3: Admin and Configuration

Longer-term product infrastructure:

- Knowledge-base upload and indexing dashboard.
- Parsing status and failed-document repair UI.
- Rebuild index button.
- LLM configuration panel.
- User permission and knowledge-base isolation.
