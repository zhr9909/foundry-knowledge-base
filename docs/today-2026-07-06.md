# Progress Update (2026-07-06)

## 1. Multi-turn Conversation Memory

### Problem
The RAG agent had no memory between turns. When the user asked "那它的疲劳性能呢？"
(it's fatigue properties?), the system couldn't resolve "它" (it) to 6061.
Search queries degenerated to generic terms like "aluminum alloy" instead of "6061".

### Solution: Manual Message Store (_message_store)

Tried three approaches before settling on the simplest:

| Approach | Status | Issue |
|----------|--------|-------|
| LangGraph MemorySaver + append_messages reducer | ❌ Failed | `progress_callback` function not msgpack-serializable |
| State history field + rewrite_query context | ✅ Working | Simple, no serialization issues |

### Architecture

```
Frontend sends query (no history needed)
  ↓
agent_chat() loads saved messages from _message_store
  ↓
initial["history"] = saved_msgs (for _rewrite_node)
  ↓
_rewrite_node → rewrite_query(query, history)
  ↓
if history:
    build context with previous user questions
    LLM resolves pronouns using history
  ↓
agent_chat() saves new messages after execution
```

### Key Files Changed

| File | Change |
|------|--------|
| `scripts/agent.py` | Added `_message_store`, `_save_msgs()`, `_load_msgs()`, history-aware `rewrite_query()` |
| `scripts/app_server.py` | Removed `thread_id` parameter from endpoints |

### Bug Fixes Made

1. **`progress_callback` serialization** (MemorySaver crash) - Removed MemorySaver, use manual store
2. **`list(None)` TypeError** (history=None crash) - Changed `list(initial.get("history", []))` to `list(initial.get("history") or [])`
3. **thread_id not accepted** (agent_chat signature mismatch) - Removed thread_id from all calls
4. **__pycache__ stale bytecode** - Multiple cache clears needed during development
