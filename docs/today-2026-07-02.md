### 管线内测试（table_rec + detection + recognition 三模型共存）
- 第13页: OCR 42 行 = **33s** ❌ (慢 15 倍)
- 第33页: 241 行 = 超时 ❌

### 根本原因
- 6GB 显存不足以让三套模型高效运行
- table_rec (~0.5GB) + detection (~0.8GB) + recognition (~0.5GB) = ~1.8GB
- 但 CUDA 上下文切换 + kernel 编译导致第一 batch 耗时 19-66s
- 独立测试时只有 OCR 模型，所以快
